API Contracts (Example)

Adjust these paths to match your actual service implementations. This UI assumes conventional REST endpoints and JSON.

  • Team Management
    • POST /api/teams → { name }
    • GET /api/teams
    • GET /api/teams/{teamId}
    • GET /api/teams/{teamId}/roster (calls Player service)
  • Player Management
    • POST /api/players → { name }
    • GET /api/players
    • POST /api/players/{playerId}/draft → { teamId }
    • POST /api/players/{playerId}/release
  • Performance Tracking
    • POST /api/competitions → { name }
    • GET /api/stats
  • Leaderboard
    • GET /api/leaderboard

Tip: Use Azure App Service app settings to store connection strings and downstream service URLs. Protect APIs with AAD, API keys, or JWT.
